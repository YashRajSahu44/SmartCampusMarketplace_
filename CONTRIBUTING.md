Contributing
============

Thanks for your interest contributing to SmartCampus. Please follow these simple guidelines.

Process
- Fork the repository and create feature branches from `main`.
- Make small, focused commits with clear messages.
- Open a pull request describing the change and link any related issues.

Code style
- Follow the existing code style. Run `npm run lint` and `npm run format` before opening a PR.
- Ensure TypeScript type checks pass: `npm run typecheck`.

Tests
- Add unit tests for new logic and run the test suite: `npm run test`.

Review
- PRs will be reviewed by maintainers. Address requested changes in a timely manner.
